# 大作业VibeCoding巨制之CommentLab

## V1版修改说明

**修改是否有效需进一步测试**

### 1. `app.py`：前端布局与交互优化

- 将原文压力测试页面改成**双栏布局**：
  - 左侧展示内容分析、风险判断和修改方向。
  - 右侧展示模拟评论区。
- 将改写对比页改成左右对照布局，分别展示原文和改写后的分析结果。
- 按区块重新组织对比页，使以下内容左右水平对齐：
  - 原文与改写文本。
  - 综合风险判断。
  - 高风险原文与修改方向。
  - 可能的误解链。
  - 原文与改写后的模拟评论区。
- **将评论区改成多级树状结构**，回复显示在父评论下方，并通过缩进和连接线体现层级关系。
- **增加逐级返回按钮**，支持返回输入页、受众确认页和原文结果页。
- 页面阶段切换后自动滚动到顶部。

**可进一步优化评论卡片，比如更加强调点赞数量，像常见的网页评论区结构那样。**

### 2. `models/schemas.py`：扩展语义和评论数据结构

为 `ContentAnalysis` 增加：

- `content_type`：识别个人观点、日常分享、通知、道歉或推荐等内容类型。
- `tone`：记录帖子的平静、轻松或强烈等语气。
- `explicit_information`：保存帖子明确表达的信息。
- `reasonable_disagreements`：记录受众可能产生的合理分歧。
- `unsupported_inferences`：记录缺少原文依据的推断。

为 `CommentAction` 和 `Comment` 增加：

- `evidence_span`：记录评论实际回应的原文或父评论片段。
- `reaction_type`：记录支持、个人感受、第一印象、质疑、调侃、动机猜测等反应类型。

这些字段使评论不再只是孤立文本，而是能够追踪“针对什么内容、以什么方式作出反应”。新增字段均设置默认值，因此旧历史数据仍能正常加载。

**目前效果不明显，具体仍需进一步测试。**

### 3. `chains/content_analysis_chain.py`：增强帖子理解能力

- 内容分析不再只识别模糊词和风险句，还会判断帖子主题、内容类型和表达语气。
- 区分帖子明确表达的内容与模型自行推断的内容。
- 识别合理意见分歧，避免把普通个人偏好错误地当成沟通危机。
- 标记无文本依据的推断，为评论 Agent 限制不合理脑补。
- DEMO_MODE 也增加了对应的本地语义分析逻辑。

**目前效果不明显，具体仍需进一步测试。**

### 4. `chains/comment_chain.py`：重构评论生成逻辑

- 正式评论流程改为基于内容语义、Persona 属性和讨论上下文生成评论。
- 不再根据三个内置案例的关键词选择固定回答。
- 每个 Persona 会先确定关注内容和反应类型，再组织评论文本。
- 不同 Persona 可以产生支持、共鸣、第一印象、理性追问、谨慎修正、动机怀疑、观望、调侃和争议化概括等不同反应。
- 减少“信息没有说完整”“建议进一步说明”等高度重复的机械表达。
- 同一 Persona 在不同轮次再次参与时，不再重复相同评论。
- 后续轮次仍会产生新的顶级评论，同时把回复分散到不同讨论分支，避免所有回复集中在一条热评下。
- Live 模型提示词增加了语义依据、反应类型、多分支和防止无依据推断的约束。
- 精简请求字段，减少每轮评论生成的 Token 消耗。

### 5. `chains/audience_chain.py`：降低受众规划请求开销

- 不再向模型发送完整内容分析对象，只发送核心表达、内容类型、语气、涉及群体和少量潜在误解。
- Persona 模板只发送 ID、标签、所属群体和简短描述。
- Persona 的完整底层参数仍由本地模板保存，模型只负责规划权重。
- 这样既减少输入 Token，也降低结构化输出请求超时的概率。

### 6. `simulation/engine.py`：支持语义驱动的多轮模拟

- 原文和改写后的模拟都会接收对应版本的内容分析结果。
- 评论 Agent 能够在每轮生成时读取帖子语义信息。
- 评论创建后保存 `evidence_span` 和 `reaction_type`。
- 保留原有固定随机种子、Persona 激活、热度排序和潜水用户点赞机制。
- 支持多个顶级评论和多个回复分支。

### 7. `services/orchestrator.py`：调整完整业务流程

- 原文模拟时传入原文的内容分析结果。
- 改写后重新分析改写稿，并把新的分析结果传入二次模拟。
- 确保两次评论模拟分别依据各自文本，而不是共用原文语义。
- 保持相同 Persona、随机种子、轮数和热度规则，保证改写前后对比公平。

### 8. `services/database.py`：扩展评论持久化信息

- 在评论内部标签中保存 `evidence_span`。
- 保存评论的 `reaction_type`。
- 保留原有项目、Persona、评论、风险报告和模型缓存结构。
- 历史项目继续通过完整 JSON 快照恢复，不需要重新调用模型。

### 9. `tests/test_simulation.py`：增加真实性相关测试

新增测试用于检查：

- 自定义帖子可以形成多个顶级讨论。
- 回复会指向多个不同的父评论。
- 评论文本不会重复。
- “建议说明”“没有说完整”等机械句式不会集中出现。
- 每条评论都具有语义依据片段。
- 评论区包含多种不同反应类型。
- 内容分析能够识别个人观点类帖子。
- 固定随机种子下结果仍然可复现。
